Recommended Reading and Platforms

A curated, opinionated list. Not everything — just the things that are actually worth your time.

Algorithm Practice Platforms

Platform Best For Honest Assessment Verdict
LeetCode
leetcode.com		 Interview prep, FAANG-level problems, pattern mastery The industry standard. Do the Blind 75 first — they cover the essential patterns. Then NeetCode 150 if you want more. Focus on understanding patterns, not memorizing solutions. The difference between candidates who pass and those who don't is usually pattern recognition, not memorization. Essential
NeetCode.io
neetcode.io		 Structured roadmap, pattern-based learning, video explanations Free, excellent, and organized by pattern rather than just difficulty. The roadmap is the best structured preparation guide available. Built by an engineer who went through the grind and documented what actually works. Start here if you're overwhelmed by LeetCode's lack of structure. Essential (free)
AlgoExpert
algoexpert.io		 Curated problem set with high-quality video explanations 160 curated problems with video walkthroughs by ex-FAANG engineers. Worth the subscription if you learn better from video and prefer quality explanations over sheer volume. The interface is clean and distraction-free. Not a replacement for LeetCode but a strong supplement. Good (paid)
HackerRank
hackerrank.com		 Beginners, automated screening tests, language familiarity Better for absolute beginners — cleaner UI, lower difficulty ceiling, used by many companies for automated screening assessments. Less depth than LeetCode for senior-level prep. If a company sends you a HackerRank test, practice here to get familiar with the environment. Situational
Codewars
codewars.com		 Language proficiency, code style, community solutions Kata-style problems with excellent community solutions to learn from. Great for sharpening language proficiency and seeing idiomatic code. Less interview-focused than the others — you won't find graph traversal problems here. Use it to maintain fluency between study sessions. Supplementary

System Design Resources

Resource Format What It's Best For Verdict
Designing Data-Intensive Applications
Martin Kleppmann		 Book The best technical book written in the last decade. Covers databases, replication, partitioning, distributed transactions, stream processing — everything a senior engineer should understand at depth. Read it cover to cover if you're targeting senior roles. Dense but genuinely accessible. No shortcuts exist for this one. Required reading for senior roles
System Design Primer
GitHub · donnemartin		 Free GitHub repo Comprehensive, covers almost everything, completely free. A good starting point and a useful reference. The main weakness is that it can feel scattered — it's a collection of topics rather than a narrative. Use it as a reference after you've built a mental model elsewhere. Good (free)
ByteByteGo
Newsletter + YouTube · Alex Xu		 Newsletter, video Alex Xu's visual explanations are some of the clearest available anywhere. The diagrams alone are worth subscribing for. Covers new topics regularly and keeps pace with the industry. The YouTube channel is free; the newsletter is a cheap paid upgrade. Subscribe to both. Excellent (mostly free)
System Design Interview Vol. 1 & 2
Alex Xu		 Book Practical case studies written in the exact format of real system design interviews. Vol. 1 is essential — 16 well-chosen cases with clear tradeoff discussions. Vol. 2 is valuable if you want more cases after completing Vol. 1. These books will not replace DDIA for depth, but they're the closest thing to interview practice in print. Essential (Vol. 1), Good (Vol. 2)

Backend Engineering Books

Book Author What It's Best For Read It If...
Clean Architecture Robert C. Martin Structuring code to be maintainable, testable, and independent of frameworks and databases. Strong opinions on dependency direction and boundaries. You want to think more clearly about how to organize large codebases and argue the tradeoffs in your next architecture discussion
Database Internals Alex Petrov Understanding how databases actually work at the implementation level — storage engines, B-trees, LSM-trees, consensus algorithms, replication internals. You want to go deep on why databases behave the way they do, not just how to use them. Not essential for most interviews, but invaluable for staff-level roles
The Pragmatic Programmer
20th Anniversary Edition		 Hunt & Thomas Professional habits, mindset, and craft — how to think about software, not just how to write it. Less about code, more about how to be effective over a career. You're early in your career and want to build good habits, or you've been doing this a while and want to articulate what separates great engineers from good ones
Release It! Michael Nygard Stability patterns for production software — circuit breakers, bulkheads, timeouts, fail-fast design. The book that put these patterns on the map for most engineers. You've been asked about resilience patterns in interviews, or you work on systems where availability matters and you want to understand why they fail

Interview Prep Books

Book Author Honest Assessment Verdict
Cracking the Coding Interview Gayle Laakmann McDowell The classic, and still worth reading. The algorithm walkthroughs are excellent — rigorous without being academic. The behavioral interview framework is solid. Some content feels dated (Java-heavy, early-Google era), but the fundamentals don't change. Read this if you're targeting FAANG or FAANG-adjacent companies. Essential for FAANG targeting
Grokking the System Design Interview Educative.io Case-study format covering common design problems (URL shortener, Instagram, Dropbox). Good breadth, very practical structure. Useful supplement to DDIA — it sacrifices depth for accessibility. Good if you learn better from structured case studies than from reading about internals. Good supplement

Language-Specific Resources

Language Resource Why This One
JavaScript "You Don't Know JS" series — Kyle Simpson (free on GitHub) The definitive deep-dive into how JavaScript actually works. Not a beginner resource — it assumes you can write JS and teaches you to understand it. Especially the Scope & Closures and this & Object Prototypes books. Required reading before any senior JavaScript interview.
TypeScript Total TypeScript — Matt Pocock (totaltypescript.com) The best TypeScript learning resource currently available. Pocock's workshops are hands-on and go well beyond "add types to JavaScript." Covers advanced generics, conditional types, and the mental models that make TypeScript actually useful at scale.
Java "Effective Java" 3rd Edition — Joshua Bloch 90 items on how to use Java correctly. A canonical text — referenced in code reviews, interviews, and architecture discussions. Read before any Java senior interview. If you've been writing Java for years without reading this, you will recognize patterns you've been using wrong.
Go "The Go Programming Language" — Donovan & Kernighan The official guide, written by Go's creators. Clear, concise, comprehensive. The language is small enough that one book covers it well. Read it once front to back — Go rewards understanding the language, not just the stdlib.
C# "C# in Depth" — Jon Skeet Goes further than any other C# resource into how the language actually works under the hood. Skeet is one of the highest-reputation Stack Overflow contributors in C# history — this is not a surface-level book. Read it if you want to understand why C# does what it does, not just what it does.
PHP Laravel Documentation — laravel.com/docs The best framework documentation in any language. Comprehensive, well-structured, filled with real examples. If you're working with PHP professionally, Laravel's docs double as a guide to modern PHP patterns. The framework itself is worth learning even if your job doesn't use it.

Staying Sharp Between Interviews

Habit Why It Works Practical Tip
1 LeetCode problem per day Consistency beats intensity. A 20-minute daily problem keeps your pattern recognition active. A weekend binge doesn't. Pick a problem from a pattern you're weak on, not an easy one you already know. Difficulty is calibration, not self-punishment.
Build and deploy a real side project Something with a real URL teaches you what tutorials never will — environment config, error handling, monitoring, and the decisions that don't exist in toy projects. Constraints make projects real. Pick one feature, ship it, get one user. Repeat. A GitHub repo no one runs is not a project.
Contribute to an open source project Reading production code at scale teaches you patterns you won't learn from courses. Even documentation PRs require you to understand the codebase. Start with issues labelled "good first issue." Understand the codebase before proposing changes. Reading is 80% of the contribution.
Write about what you learn Writing forces clarity. If you cannot explain a concept simply and precisely in prose, you don't understand it well enough to explain it in an interview. Write for a specific audience — a teammate who wasn't there, a past version of yourself. Publish or don't, but write.
Follow engineering blogs Real companies write about real problems at scale. Reading what Netflix, Cloudflare, or Uber actually built teaches system design better than any hypothetical. Netflix Tech Blog, Uber Engineering, Cloudflare Blog, Martin Fowler's bliki, The Pragmatic Engineer (newsletter). Read one post per week — understanding it deeply beats skimming ten.

You now have everything you need. The gap between where you are and where you want to be is not knowledge — it's practice. Put down the book. Open your terminal. Write something. The interview is just the exam. The real work is what you build in between.

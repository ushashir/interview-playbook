Backend Question Taxonomy

A classified list of 70+ real interview questions, organized by category with difficulty ratings and key concepts to address.

How to use this page: Scan for questions you can't answer confidently. Go to the chapter listed and study that section. Return here and answer the question out loud — without notes — until it flows naturally.

API Design — Ch 17

Question Difficulty Key Concept to Address Ch
What's the difference between PUT and PATCH? Beginner Idempotency; PUT replaces the full resource, PATCH applies a partial update 17
How would you design a rate-limited API? Intermediate Token bucket or sliding window algorithm; Redis counter; HTTP 429 response; retry-after header 17
Explain REST constraints. Intermediate Stateless, cacheable, uniform interface, layered system, client-server, code on demand (optional) 17
How do you version a REST API? Intermediate URL path (/v1/), Accept header, query param; tradeoffs of each; deprecation strategy 17
When would you use GraphQL over REST? Intermediate Over-fetching / under-fetching; multiple client types with different data needs; schema-first development 17
What is idempotency and why does it matter? Intermediate Same request, same result no matter how many times called; safe retries; POST idempotency key pattern 17
How do you design a REST API for a multi-tenant SaaS? Advanced Tenant isolation (subdomain vs header vs path); authorization checks on every resource; row-level security 17
Explain the Richardson Maturity Model. Advanced Level 0 (one URI); Level 1 (resources); Level 2 (HTTP verbs); Level 3 (HATEOAS hypermedia) 17

Databases — Ch 18

Question Difficulty Key Concept to Address Ch
What is a database index? Beginner B-tree structure; speeds up reads at the cost of write overhead and storage; when not to add one 18
What is the N+1 query problem? Beginner 1 query to fetch a list + N queries for related data; fix with eager loading, JOIN, or DataLoader 18
Explain ACID properties. Intermediate Atomicity (all or nothing), Consistency (valid state), Isolation (concurrent tx don't interfere), Durability (persists after crash) 18
When would you choose NoSQL over SQL? Intermediate Flexible/evolving schema, horizontal write scale, access patterns that don't need JOINs, document or key-value model 18
How do you optimize a slow query? Intermediate EXPLAIN ANALYZE; check for missing index; covering index; avoid SELECT *; rewrite with CTE or subquery 18
Explain database normalization. Intermediate 1NF (atomic values), 2NF (no partial dependency), 3NF (no transitive dependency); denormalization for read performance 18
How do you design a Redis caching layer? Intermediate Cache-aside pattern; key naming convention; TTL strategy; cache invalidation on write; stampede protection 18
What is database sharding? Advanced Horizontal partitioning; shard key selection; hotspot risk; cross-shard queries; resharding complexity 18

Distributed Systems — Ch 19

Question Difficulty Key Concept to Address Ch
What is CAP theorem? Intermediate Consistency, Availability, Partition Tolerance — can only guarantee two; CP vs AP systems; real-world examples (Zookeeper = CP, Cassandra = AP) 19
Explain the Saga pattern. Intermediate Distributed transactions via sequence of local transactions; choreography (events) vs orchestration (central coordinator); compensating transactions 19
When would you NOT use microservices? Intermediate Small team, early-stage product, unclear domain boundaries, operational complexity not justified; Conway's Law implication 19
What is event sourcing? Advanced Store all state changes as immutable events; derive current state by replaying; audit trail; pairs naturally with CQRS; projection challenges 19
Explain the Outbox pattern. Advanced Solves dual-write: write event to outbox table in same DB transaction; separate relay process publishes to message broker; CDC as alternative 19
How do you handle distributed transactions? Advanced 2PC vs Saga; why 2PC is impractical (blocking, coordinator as SPOF); Saga preferred; eventual consistency acceptance 19
What is consistent hashing and why is it useful? Advanced Hash ring maps keys to nodes; adding/removing a node only remaps adjacent keys; virtual nodes for better distribution; used in distributed caches 19

Performance & Observability — Ch 20

Question Difficulty Key Concept to Address Ch
What is P99 latency? Beginner 99th percentile; 99% of requests are faster; tail latency; why mean/average hides bad user experiences 20
What are SLO, SLA, and SLI? Intermediate SLI = measured indicator; SLO = internal target; SLA = contractual agreement with penalty; error budget = 1 - SLO 20
How do you debug a production outage? Intermediate Check dashboards → recent deploys → infrastructure metrics → logs → isolate to service/endpoint → fix or rollback 20
What are the three pillars of observability? Intermediate Logs (events), Metrics (aggregated numbers over time), Traces (request flow across services); each answers different questions 20
How do you implement graceful shutdown? Intermediate Listen for SIGTERM; stop accepting new connections; drain in-flight requests; close DB connections; exit cleanly 20
What causes memory leaks in Node.js? Intermediate Unremoved event listeners, closures holding references, unbounded global caches, missing cleanup in setInterval; detection with heap snapshots 20
How do you reduce P99 latency? Advanced Identify tail causes (GC pause, lock contention, cold cache); hedged requests; async where possible; caching; timeout budgets 20

Security — Ch 22

Question Difficulty Key Concept to Address Ch
What's the difference between authentication and authorization? Beginner Authentication = who are you (identity); Authorization = what are you allowed to do (access control) 22
Why shouldn't you store passwords in plain text? Beginner Data breach exposes all passwords; rainbow table attacks; use bcrypt/Argon2 with per-user salt 22
What is SQL injection and how do you prevent it? Beginner Attacker injects SQL via user input; prevent with parameterized queries / prepared statements; ORM helps but doesn't eliminate 22
What is JWT and how does it work? Intermediate Header.Payload.Signature; server signs with secret; stateless auth; token expiry; refresh token pattern; never store sensitive data in payload 22
Explain XSS and how to prevent it. Intermediate Attacker injects malicious script into page viewed by other users; prevent with output escaping, CSP headers, textContent vs innerHTML 22
What is CSRF and how do you prevent it? Intermediate Forged cross-site request using victim's session; prevent with SameSite cookie attribute, CSRF token in forms/headers 22
What is SSRF? Advanced Server-Side Request Forgery; attacker makes server fetch internal URLs (metadata endpoints, internal services); prevent with URL allowlist, block private IP ranges 22
How would you secure a multi-tenant API? Advanced Verify tenant ownership on every resource access; row-level security in DB; never trust tenant ID from client alone; audit log all sensitive operations 22

System Design — Ch 29–32

Question Difficulty Key Concept to Address Ch
Design a URL shortener. Intermediate Base62 hash or counter; redirect (301 vs 302); scale reads with cache; analytics via async events; custom aliases 31
Design a notification service. Intermediate Queue-based fanout; channel routing (push/email/SMS); retry with exponential backoff; idempotency; preference management 31
Design a rate limiter. Intermediate Token bucket or sliding window log; Redis for distributed counter; per-user vs per-IP vs per-endpoint; HTTP 429 + retry-after 30
Design a product search system. Advanced Elasticsearch as read store; CDC sync from primary DB; relevance scoring; faceted filters; autocomplete; search latency SLO 31
Design a real-time chat application. Advanced WebSockets for bidirectional connection; message ordering guarantees; persistence in DB; fan-out for group chats; presence indicators 31
Design a distributed cache. Advanced Consistent hashing for key distribution; eviction policies (LRU, LFU); replication for availability; cache coherence strategy 30

Behavioral — Ch 1–3

PHP Deep Dive

PHP powers WordPress (43% of the web), Slack's early backend, and Facebook's origins. Its reputation is a decade out of date. PHP 8.x is a genuinely modern language — and if you're interviewing at agencies, media companies, or traditional web shops, PHP questions will come.

What you'll learn

  • PHP 8.x features that separate modern PHP from its legacy reputation
  • Laravel patterns: Eloquent ORM, service container, queues, middleware
  • The type juggling trap — == vs === and why it costs jobs
  • Composer, PSR-4 autoloading, and dependency management
  • Performance tuning: OPcache, JIT, and connection pooling
  • Array functions, closures, generators, traits, and anonymous classes

PHP 8.x — What Makes Modern PHP Different

Analogy: PHP is like a city that grew organically over 30 years — some neighborhoods are old and messy, some are brand new and beautiful. The key is knowing which neighborhood you're in. Modern PHP 8.x with Laravel is like a newly built district — clean streets, well-planned infrastructure. Legacy PHP 4 code from 2002 is a different city entirely. Most interviewers want to know: do you know the new part of town?

Named Arguments (PHP 8.0)

Named arguments let you pass values by parameter name, skipping positional order and making calls self-documenting. They also let you skip optional parameters you don't care about.

<?php

// Without named arguments — you have to look up what true, 0, false mean
$result = array_slice($array, 0, 5, true);

// With named arguments — self-documenting
$result = array_slice(array: $array, offset: 0, length: 5, preserve_keys: true);

// Skip optional parameters without passing placeholders
function createUser(
    string $name,
    string $email,
    string $role = 'user',
    bool   $active = true,
    ?string $avatar = null
): User {
    // ...
}

// Only set what you need — no need to pass 'user', true, null
$user = createUser(name: 'Alice', email: 'alice@example.com', active: false);

Match Expression (PHP 8.0)

Analogy: match is switch that grew up. It's exhaustive (throws if no arm matches), returns a value (no assignment needed inside), uses strict comparison (=== not ==), and has no fall-through. It's what switch always should have been. 
<?php

$status = 'shipped';

// Old switch — verbose, fall-through risk, == comparison
switch ($status) {
    case 'pending':
        $label = 'Pending';
        break;
    case 'shipped':
        $label = 'Shipped';
        break;
    default:
        $label = 'Unknown';
}

// New match — concise, strict, returns value, throws UnhandledMatchError on no match
$label = match($status) {
    'pending'    => 'Pending',
    'shipped',
    'delivered'  => 'On its way',    // multiple arms share a result
    'cancelled'  => 'Cancelled',
    default      => throw new \InvalidArgumentException("Unknown status: $status"),
};

// match with no-subject form (PHP 8.0)
$discount = match(true) {
    $total > 1000 => 0.20,
    $total > 500  => 0.10,
    $total > 100  => 0.05,
    default       => 0.00,
};

Enums (PHP 8.1)

<?php

// Pure enum — no backing value
enum Direction {
    case North;
    case South;
    case East;
    case West;
}

// Backed enum — each case has a scalar value (string or int)
enum OrderStatus: string
{
    case Pending   = 'pending';
    case Shipped   = 'shipped';
    case Delivered = 'delivered';
    case Cancelled = 'cancelled';

    // Enums can have methods
    public function label(): string
    {
        return match($this) {
            self::Pending   => 'Awaiting processing',
            self::Shipped   => 'On its way',
            self::Delivered => 'Delivered',
            self::Cancelled => 'Cancelled',
        };
    }

    public function isFinal(): bool
    {
        return in_array($this, [self::Delivered, self::Cancelled]);
    }
}

// Usage
$status = OrderStatus::from('shipped');       // throws ValueError if not found
$status = OrderStatus::tryFrom('unknown');    // returns null if not found

echo $status->value;         // 'shipped'
echo $status->label();       // 'On its way'
echo $status->name;          // 'Shipped'

// Enums can implement interfaces
interface HasColor {
    public function color(): string;
}

enum Priority: int implements HasColor
{
    case Low    = 1;
    case Medium = 2;
    case High   = 3;

    public function color(): string
    {
        return match($this) {
            self::Low    => 'green',
            self::Medium => 'yellow',
            self::High   => 'red',
        };
    }
}

Nullsafe Operator, Union Types, and Fibers

<?php

// Nullsafe operator (PHP 8.0) — short-circuits on null
// Before:
$avatar = null;
if ($user !== null && $user->profile !== null) {
    $avatar = $user->profile->avatar;
}

// After:
$avatar = $user?->profile?->avatar;  // null if any step is null

// Chain method calls too
$country = $order?->getCustomer()?->getAddress()?->getCountry();

// Union types (PHP 8.0)
function processId(int|string $id): int|string
{
    return is_string($id) ? (int) ltrim($id, '#') : $id;
}

// Fibers (PHP 8.1) — cooperative multitasking / coroutines
$fiber = new Fiber(function (): void {
    $value = Fiber::suspend('first');   // pause, yield 'first' to caller
    echo "Fiber received: $value\n";
    Fiber::suspend('second');
});

$result1 = $fiber->start();            // runs until first suspend
echo "Got from fiber: $result1\n";     // 'first'

$result2 = $fiber->resume('hello');   // resumes, passes 'hello' in
echo "Got from fiber: $result2\n";     // 'second'
// Fibers are the foundation for async PHP libraries like ReactPHP and Revolt

Laravel Patterns

Analogy: Laravel is to PHP what Spring Boot is to Java, or NestJS is to Node.js — an opinionated, batteries-included framework. Learning its patterns is more important than raw PHP trivia in Laravel-shop interviews.

Eloquent ORM

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\HasMany;
use Illuminate\Database\Eloquent\Relations\BelongsTo;

class User extends Model
{
    protected $fillable = ['name', 'email', 'role'];
    protected $hidden   = ['password', 'remember_token'];
    protected $casts    = [
        'email_verified_at' => 'datetime',
        'is_admin'          => 'boolean',
    ];

    public function orders(): HasMany
    {
        return $this->hasMany(Order::class);
    }
}

class Order extends Model
{
    protected $fillable = ['user_id', 'amount', 'status'];
    protected $casts    = ['amount' => 'decimal:2'];

    public function user(): BelongsTo
    {
        return $this->belongsTo(User::class);
    }

    // Local scope — reusable query constraint
    public function scopeShipped($query)
    {
        return $query->where('status', 'shipped');
    }
}

// Fluent query builder
$users = User::where('active', true)
    ->orderBy('name')
    ->paginate(20);                          // SELECT ... LIMIT 20 OFFSET ...

// Eager loading — avoid N+1
$orders = Order::with(['user', 'items'])    // one query for orders, one for users, one for items
    ->shipped()                             // local scope
    ->whereBetween('created_at', [now()->subMonth(), now()])
    ->get();

// Chunking — process large datasets without loading all into memory
Order::where('status', 'pending')
    ->chunk(100, function ($orders) {
        foreach ($orders as $order) {
            ProcessOrder::dispatch($order);
        }
    });

Service Container and Dependency Injection

<?php

// Laravel's service container resolves dependencies automatically
// Just type-hint in constructors and Laravel injects them

class OrderController extends Controller
{
    public function __construct(
        private readonly OrderService     $orderService,
        private readonly PaymentGateway   $payments,

    {
        // $request is a Form Request — validation runs before this method

    }

// Bind an interface to a concrete implementation in AppServiceProvider
public function register(): void
{
    $this->app->bind(
        PaymentGateway::class,
        fn() => new StripeGateway(config('services.stripe.secret'))

    // Singleton — same instance for the whole request lifecycle

{

    {
        return [
            'amount'   => ['required', 'numeric', 'min:0.01'],
            'items'    => ['required', 'array', 'min:1'],

            'items.*.quantity'   => ['required', 'integer', 'min:1'],

Queues and Jobs

<?php

// Create a job: php artisan make:job ProcessOrderJob
class ProcessOrderJob implements ShouldQueue
{
    use Dispatchable, InteractsWithQueue, Queueable, SerializesModels;

    public int $tries = 3;
    public int $backoff = 60;  // seconds between retries

    public function __construct(
        public readonly Order $order
    ) {}

    public function handle(NotificationService $notifications): void
    {
        // Runs asynchronously in a queue worker process
        $this->order->process();
        $notifications->sendConfirmation($this->order);
    }

    public function failed(\Throwable $exception): void
    {
        // Called when all retries exhausted
        Log::error('Order processing failed', [
            'order_id' => $this->order->id,
            'error'    => $exception->getMessage(),
        ]);
    }
}

// Dispatch — push to queue (Redis, database, SQS, etc.)
ProcessOrderJob::dispatch($order);
ProcessOrderJob::dispatch($order)->delay(now()->addMinutes(5));
ProcessOrderJob::dispatch($order)->onQueue('critical');

// Run workers: php artisan queue:work redis --queue=critical,default

The Type Juggling Trap — == vs ===

This is the most dangerous PHP feature for interview candidates. PHP's loose comparison operator == performs type coercion before comparing. The results are counterintuitive enough to trip up experienced developers. Always use === (strict equality) in professional code.
Analogy: Using == in PHP is like asking "is this document approximately the same as that one?" PHP tries to interpret both sides as the same type, then compares. Using === is asking "are these identical in both value AND type?" — which is almost always what you want. 
<?php

// Dangerous == results that trip up candidates
var_dump(0    == "foo");    // TRUE  — "foo" cast to int = 0, then 0 == 0
var_dump(0    == "");       // TRUE  — "" cast to int = 0 in PHP 7 (FALSE in PHP 8!)
var_dump(0    == "0");      // TRUE  — "0" cast to int = 0
var_dump("1"  == "01");     // TRUE  — both cast to int
var_dump("1"  == "1.0");    // TRUE  — both cast to float
var_dump(100  == "1e2");    // TRUE  — "1e2" = 100.0 in scientific notation
var_dump(""   == false);    // TRUE  — both falsy
var_dump("0"  == false);    // TRUE  — "0" is falsy
var_dump(null == false);    // TRUE
var_dump(null == "");       // TRUE
var_dump(1    == true);     // TRUE
var_dump(0    == false);    // TRUE

// None of these are true with ===
var_dump(0    === "foo");   // false — different types
var_dump(""   === false);   // false
var_dump(null === false);   // false

// Security implication — hash comparison vulnerability (PHP 7 and earlier)
// Magic hash: MD5 of "240610708" starts with "0e" — looks like 0 * 10^something
$hash = md5("240610708");   // "0e462097431906509019562988736854"
var_dump($hash == 0);       // TRUE in PHP 7! (0e... cast to 0.0 == 0)
var_dump($hash === 0);      // false — correct

// Type coercion in arithmetic context
var_dump("5 apples" + 3);   // 8 — PHP reads leading integer, ignores rest
var_dump("apples" + 3);     // 3 — "apples" coerces to 0

// The null coalescing operators — safe defaults
$value = $data['key'] ?? 'default';           // returns $data['key'] if set and not null
$data['key'] ??= 'default';                   // assigns 'default' only if null/missing

// Correct string-to-int comparison
$idFromQuery = "42";  // comes from URL or form
if ($idFromQuery === (string) $user->id) { /* correct */ }
if ((int) $idFromQuery === $user->id)    { /* correct */ }
if ($idFromQuery == $user->id)           { /* works but fragile */ }
PHP 8 changed some of these. In PHP 8, 0 == "foo" is now false (non-numeric strings no longer cast to 0 when compared with integers). But this is a nuance — the only safe rule is: always use === for comparisons, always use strict types at the top of your files (declare(strict_types=1);).

Composer and Autoloading

Analogy: Composer is PHP's npm. composer.json is your package.json, composer.lock is your package-lock.json. PSR-4 is the naming convention that maps namespaces to directory paths — the PHP equivalent of ES module resolution.

{
    "name": "yourcompany/your-service",

    "require-dev": {
        "phpunit/phpunit": "^11.0",

    "autoload": {

            "App\\": "app/",

            "Tests\\": "tests/"

PHP Performance — OPcache, JIT, and Connection Pooling

Analogy: Without OPcache, PHP reads the source file, tokenizes it, parses it, and compiles it to bytecode on every single request — like reprinting a recipe card every time you cook the same dish. OPcache caches the compiled bytecode in shared memory so the compilation step is skipped entirely on subsequent requests.

OPcache — Always Enable in Production

; php.ini — production OPcache settings
opcache.enable=1
opcache.memory_consumption=256        ; MB of shared memory for bytecode cache
opcache.interned_strings_buffer=16    ; MB for string deduplication
opcache.max_accelerated_files=20000   ; max number of cached files
opcache.validate_timestamps=0         ; DISABLE in production (use deploy hook to clear)
opcache.revalidate_freq=0             ; only matters if validate_timestamps=1
opcache.save_comments=1               ; needed for Doctrine/Laravel annotations

; Clear OPcache after deployment
; opcache_reset() in a deploy script, or restart PHP-FPM

JIT (PHP 8.0+)

; php.ini — JIT settings (PHP 8.0+)
opcache.jit=tracing         ; best for most workloads (tracing JIT)
opcache.jit_buffer_size=128M

; JIT helps most with CPU-bound workloads: image processing, math, parsers
; JIT helps LITTLE with typical I/O-bound web apps (most time is in DB/network)
; Measure before enabling — for web apps, OPcache alone is the big win

Connection Pooling

PHP has a fundamental challenge: each request spawns a new PHP-FPM worker process. Each process opens its own database connection. With 500 concurrent workers, you have 500 open connections to Postgres — which can overwhelm the database (default max_connections is 100).

The solution is a connection pooler that sits between PHP-FPM and the database:

ToolUse caseMode
PgBouncerPostgreSQLTransaction pooling (best for PHP)
ProxySQLMySQL / MariaDBQuery routing + pooling
RDS ProxyAWS RDS / AuroraManaged pooling
 
# PgBouncer config — pgbouncer.ini
[databases]
mydb = host=postgres-primary port=5432 dbname=mydb

[pgbouncer]
pool_mode = transaction         # connection returned to pool after each transaction
max_client_conn = 1000          # PHP-FPM workers can connect
default_pool_size = 20          # actual DB connections: 20 total
min_pool_size = 5
server_idle_timeout = 600

# Result: 1000 PHP workers share 20 actual Postgres connections
# PHP connects to PgBouncer on port 5432 — completely transparent

Common PHP Interview Questions

Array Functions

<?php

$numbers = [1, 2, 3, 4, 5];
$orders  = [
    ['id' => 1, 'amount' => 100],
    ['id' => 2, 'amount' => 50],
    ['id' => 3, 'amount' => 200],
];

// array_map — transform each element, returns new array
$doubled = array_map(fn($n) => $n * 2, $numbers);         // [2,4,6,8,10]
$amounts = array_map(fn($o) => $o['amount'], $orders);     // [100,50,200]

// array_filter — keep elements matching predicate
$even     = array_filter($numbers, fn($n) => $n % 2 === 0); // [1=>2, 3=>4]
// NOTE: array_filter preserves keys — use array_values() to reindex

// array_reduce — fold to a single value
$total = array_reduce($orders, fn($carry, $o) => $carry + $o['amount'], 0); // 350

// usort — sort by custom comparator
usort($orders, fn($a, $b) => $a['amount'] <=> $b['amount']);  // <=> is spaceship operator

// array_column — extract a column (like SQL SELECT column)
$ids = array_column($orders, 'id');                             // [1,2,3]
$byId = array_column($orders, null, 'id');                     // indexed by id

// array_combine — zip keys and values
$keys   = ['name', 'email', 'role'];
$values = ['Alice', 'alice@example.com', 'admin'];
$user = array_combine($keys, $values);  // ['name'=>'Alice', ...]

Closures with use Keyword

<?php

$multiplier = 3;

// Capture by value — $multiplier is copied at definition time
$times3 = function(int $n) use ($multiplier): int {
    return $n * $multiplier;
};

$multiplier = 10;  // this change has NO effect on $times3
echo $times3(5);   // 15 — uses the captured value of 3

// Capture by reference — changes to $multiplier ARE reflected
$times3ref = function(int $n) use (&$multiplier): int {
    return $n * $multiplier;
};

echo $times3ref(5);  // 50 — uses current value of $multiplier (10)

// Arrow functions (PHP 7.4+) — auto-capture by value, single expression
$times3arrow = fn(int $n): int => $n * $multiplier;
// Note: arrow functions capture all outer variables automatically (by value)

Generators

<?php

// Generator — lazy iteration, processes one item at a time, no memory spike
function readLargeFile(string $path): \Generator
{
    $handle = fopen($path, 'r');
    if ($handle === false) {
        throw new \RuntimeException("Cannot open: $path");
    }

    try {
        while (($line = fgets($handle)) !== false) {
            yield trim($line);  // pause here, return line to caller
        }
    } finally {
        fclose($handle);  // always close, even if generator is abandoned
    }
}

// Process a 2GB CSV without loading it all into memory
foreach (readLargeFile('/data/orders.csv') as $line) {
    $row = str_getcsv($line);
    processRow($row);
}

// Generator with keys
function indexedItems(array $items): \Generator
{
    foreach ($items as $index => $item) {
        yield $index => strtoupper($item);
    }
}

// Send values INTO a generator
function accumulator(): \Generator
{
    $total = 0;
    while (true) {
        $value = yield $total;  // yield current total, receive next value via send()
        if ($value === null) break;
        $total += $value;
    }
}

$gen = accumulator();
$gen->current();       // initialize
$gen->send(10);        // send 10, total = 10
$gen->send(25);        // send 25, total = 35
echo $gen->current();  // 35

Traits

<?php

// Trait — horizontal code reuse (PHP has no multiple inheritance)
trait Timestamps
{
    private \DateTimeImmutable $createdAt;
    private \DateTimeImmutable $updatedAt;

    public function touch(): void
    {
        $this->updatedAt = new \DateTimeImmutable();
    }

    public function getCreatedAt(): \DateTimeImmutable
    {
        return $this->createdAt;
    }
}

trait SoftDeletes
{
    private ?\DateTimeImmutable $deletedAt = null;

    public function softDelete(): void
    {
        $this->deletedAt = new \DateTimeImmutable();
    }

    public function isDeleted(): bool
    {
        return $this->deletedAt !== null;
    }
}

class Order
{
    use Timestamps, SoftDeletes;  // compose both traits

    public function __construct(
        private readonly int $id,
        private float $amount,
    ) {
        $this->createdAt = new \DateTimeImmutable();
        $this->updatedAt = new \DateTimeImmutable();
    }
}

Senior vs Junior Comparison

Junior answer — "What is the difference between == and === in PHP?"

"Double equals checks value, triple equals checks value and type. So 1 == "1" is true but 1 === "1" is false."

Senior answer

"Triple equals (===) is strict identity — both operands must be the same type AND have the same value. Double equals (==) coerces operands to a common type first, following PHP's type juggling rules — and those rules are genuinely dangerous.

The most critical case in production code is the so-called 'magic hash' vulnerability in PHP 7: strings starting with 0e followed by digits are interpreted as floating-point numbers in scientific notation, so two completely different MD5 hashes starting with 0e would be considered equal under ==. This breaks hash comparison authentication.

In practice, I enforce declare(strict_types=1) in every file — this makes all native type coercions explicit failures rather than silent — and I use === everywhere. The only time == is defensible is comparing a known integer to a value you've already validated as numeric. But at that point you've already cast it, so you're using === anyway.

With PHP 8, some of the worst cases changed — 0 == "foo" is now false — but the rules are still non-obvious enough that relying on == anywhere is technical debt."

Practice Questions

Beginner

Q: What does declare(strict_types=1) do in PHP, and why should you always include it?

A: It enables strict type checking for scalar types in that file. Without it (in the default "coercive mode"), PHP silently converts a string "5" to int 5 when passed to a function declaring int $id. With strict types enabled, passing a string to an int parameter throws a TypeError immediately — the bug surfaces at call time rather than producing wrong results silently. It must be the very first statement in the file (before namespace declarations). Using it consistently across your codebase makes type errors loud and early, which is always better than silent coercion in production code.

Intermediate

Q: Explain the N+1 query problem in Eloquent and how to fix it.

A: The N+1 problem occurs when you load a collection of N models and then access a relationship on each — triggering one query to load the collection, plus one query per row to load the related model (N more). For example: $orders = Order::all(); foreach ($orders as $order) { echo $order->user->name; } fires 1 query for orders + 1 query per order for the user = 1+N queries. Fix with eager loading: $orders = Order::with('user')->get(); which fires exactly 2 queries regardless of N — one for orders, one to load all related users using WHERE id IN (...). For nested relationships: Order::with('user.address')->get(). In production, use Laravel Telescope or a query logger in development to catch N+1 issues before they reach production.

Advanced

Q: Implement a type-safe pipeline pattern in PHP 8.2 where each stage transforms data and the output type of one stage must match the input type of the next. Include error handling.

<?php

declare(strict_types=1);

namespace App\Pipeline;

/**
 * A Stage transforms data of type TIn to TOut.
 * PHP can't enforce generic type constraints at runtime, so we use
 * the type system where we can and document the contract clearly.
 *
 * @template TIn
 * @template TOut
 */
interface Stage
{
    /** @param TIn $data */
    public function process(mixed $data): mixed;
}

/**
 * @template T
 */
final class Pipeline
{
    /** @var list<Stage> */
    private array $stages = [];

    public static function send(mixed $data): self
    {
        $pipeline = new self();
        $pipeline->passable = $data;
        return $pipeline;
    }

    private mixed $passable;

    public function through(Stage ...$stages): self
    {
        $this->stages = array_values($stages);
        return $this;
    }

    public function thenReturn(): mixed
    {
        $data = $this->passable;
        foreach ($this->stages as $index => $stage) {
            try {
                $data = $stage->process($data);
            } catch (\Throwable $e) {
                throw new PipelineException(
                    message: sprintf(
                        'Pipeline failed at stage %d (%s): %s',
                        $index,
                        $stage::class,
                        $e->getMessage()
                    ),
                    stage: $index,
                    stageName: $stage::class,
                    previous: $e
                );
            }
        }
        return $data;
    }
}

final class PipelineException extends \RuntimeException
{
    public function __construct(
        string             $message,
        public readonly int    $stage,
        public readonly string $stageName,
        ?\Throwable        $previous = null
    ) {
        parent::__construct($message, 0, $previous);
    }
}

// --- Concrete stages for an order processing pipeline ---

final class ValidateOrder implements Stage
{
    public function process(mixed $data): array
    {
        if (!is_array($data)) {
            throw new \InvalidArgumentException('Expected array order data');
        }
        if (empty($data['items'])) {
            throw new \DomainException('Order must have at least one item');
        }
        if (($data['amount'] ?? 0) <= 0) {
            throw new \DomainException('Order amount must be positive');
        }
        return $data;
    }
}

final class EnrichWithCustomer implements Stage
{
    public function __construct(
        private readonly CustomerRepository $customers
    ) {}

    public function process(mixed $data): array
    {
        $customer = $this->customers->find($data['customer_id'])
            ?? throw new \RuntimeException("Customer {$data['customer_id']} not found");

        return [...$data, 'customer' => $customer];
    }
}

final class ApplyDiscounts implements Stage
{
    public function process(mixed $data): array
    {
        $discount = match(true) {
            $data['amount'] > 1000 => 0.20,
            $data['amount'] > 500  => 0.10,
            $data['amount'] > 100  => 0.05,
            default                 => 0.00,
        };

        return [
            ...$data,
            'discount'        => $discount,
            'discounted_amount' => round($data['amount'] * (1 - $discount), 2),
        ];
    }
}

// Usage
try {
    $processedOrder = Pipeline::send($rawOrderData)
        ->through(
            new ValidateOrder(),
            new EnrichWithCustomer($customerRepo),
            new ApplyDiscounts(),
        )
        ->thenReturn();
} catch (PipelineException $e) {
    Log::error('Order pipeline failed', [
        'stage'     => $e->stage,
        'stageName' => $e->stageName,
        'error'     => $e->getMessage(),
        'cause'     => $e->getPrevious()?->getMessage(),
    ]);
    throw $e;
}

Key design decisions: Each stage is a small, testable class with a single responsibility — you can unit test ValidateOrder in complete isolation. PHP's type system can't enforce generic constraints at runtime, but PHPDoc generics (@template) give IDEs and static analysis tools (Psalm, PHPStan) enough information to flag type mismatches at development time. The PipelineException wraps the original exception while adding which stage failed — critical for debugging pipelines in production logs. The spread operator [...$data, 'key' => 'value'] creates immutable-style transformations without mutating the original array.

Chapter 28 Summary

  • Modern PHP 8.x is a fundamentally different language from its legacy reputation: named arguments, match expressions, enums, fibers, constructor promotion, and nullsafe operators make it expressive and safe when used correctly.
  • Laravel's ecosystem (Eloquent, service container, queues, form requests) follows the same DI and ORM patterns as Spring Boot and NestJS — understanding those patterns transfers directly.
  • Always use === and declare(strict_types=1) — PHP's type juggling with == produces genuinely dangerous results and has caused real security vulnerabilities; this is a guaranteed interview question in PHP shops.
  • OPcache is the single biggest PHP performance win — it caches compiled bytecode so PHP never re-parses source files on production traffic; always enable it and clear it as part of your deployment pipeline.
  • Generators solve the large-dataset memory problem cleanly, connection pooling (PgBouncer/ProxySQL) solves the "500 workers = 500 DB connections" problem, and eager loading with with() solves the N+1 query problem — knowing all three shows production-grade PHP thinking.
